## 时间复杂度



~~~
时间复杂度：是一个衡量算法执行时间与输入数据规模之间关系的度量。它描述了算法随着输入规模的增加，其执行时间是如何变化的。通常情况下，我们关注的是最坏情况下的时间复杂度。
我们用大 O 表示法
~~~

### 时间复杂度的常见分类

#### 常数时间复杂度：O(1)

无论输入数据的规模如何变化，算法执行的时间始终保持不变。
例如：访问数组的一个元素。

```cpp
int arr[10];
int x = arr[5];  // O(1)



#### 线性时间复杂度：O(n)
算法的执行时间与输入数据的规模n成正比。
例如：遍历一个数组。

```cpp
for (int i = 0; i < n; i++) {
    // O(n)
}
```



#### 平方时间复杂度：O(n^2)
执行时间与输入数据规模的平方成正比。
例如：两个嵌套的循环，遍历一个二维数组。
```cpp
for (int i = 0; i < n; i++) {
    for (int j = 0; j < n; j++) {
        // O(n^2)
    }
}
```


#### 对数时间复杂度：O(log n)
执行时间随着输入规模增加而增长较慢，通常出现在分治法算法中。
例如：二分查找。

```cpp
while (l < r) {
    int mid = (l + r + 1) / 2;
    if (check(mid)) l = mid;
    else r = mid - 1;
}
```

#### 线性对数时间复杂度：O(n log n)
常见于一些高效的排序算法，如归并排序、快速排序。

~~~
std::sort(...);
~~~


### 分析时间复杂度 

在算法题分析中，首先**分析时间复杂度** 是一个非常好的习惯 —— 这能帮助你快速判断算法是否可行、是否需要优化。



| 数据规模        | 可接受复杂度            | 说明         |
| ----------- | ----------------- | ---------- |
| n ≤ 10³     | O(n²) 或更低         | 小数据，可以暴力   |
| n ≤ 10⁵     | O(n log n) 或 O(n) | 中等规模，要高效算法 |
| n ≤ 10⁶     | O(n) 最多           |            |
| n ≤ 10⁷~10⁸ | O(1) 或非常接近常数时间    |            |

**如何分析？**

~~~
明确输入规模（n）
首先看题目：
“输入 n 个数” → 规模就是 n；
“n 行 m 列矩阵” → 规模是 n×m；
有嵌套循环？那就要看每层循环的取值范围。

只保留增长最快的部分。
例如：
O(n² + n) → O(n²)
O(3n) → O(n)
~~~


## 空间复杂度

~~~
空间复杂度 表示程序在运行过程中 所占用的内存空间 随输入规模 𝑛 的变化关系。
和时间复杂度一样，我们关注的是 渐进增长趋势，即大O表示法。
~~~

O(1): int a;
O(n); int a[n];
O(n^2) int a[n][n];


**时间复杂度看“循环”，空间复杂度看“数组”。**



## 排序


在算法竞赛中，排序是最常见的基础操作之一，几乎出现在所有类型的题目中。  
无论是贪心、二分、离散化、区间合并、扫描线还是动态规划，排序都经常是第一步。

---

### 常用排序方式（竞赛视角）

| 排序方式 | 使用场景 | 复杂度 | 说明 |
|-----------|------------|------------|-----------|
| `std::sort()` | 通用排序 | O(n log n) | 快速、稳定、支持自定义比较 |
| `std::stable_sort()` | 需要稳定性时 | O(n log n) | 保持相等元素相对顺序 |
| `std::partial_sort()` | 只取前 k 个最小值 | O(n log k) | 常用于 Top-K 问题 |
| `std::nth_element()` | 找第 k 小值（不完全排序） | O(n) 平均 | 比完整排序更快 |
| 手写排序 | 理论学习    | - | 比赛中一般不推荐 |

> 💡 一般算法题中直接用 `std::sort()`，因为它底层是 **introsort**（快排 + 堆排 + 插排混合），性能非常优秀。

---

### 手写排序算法（了解原理即可）

#### 1. 冒泡排序（Bubble Sort）

逐对比较相邻元素，把较大的“冒泡”到后面。  
虽然时间复杂度高，但非常适合初学者理解排序的本质。

```cpp
for (int i = 0; i < n - 1; i++) {
    for (int j = 0; j < n - i - 1; j++) {
        if (a[j] > a[j + 1]) swap(a[j], a[j + 1]);
    }
}
```

#### 2. 快速排序 (Quick Sort)

```cpp
void quickSort(vector<int>& a, int l, int r) {
    if (l >= r) return;
    int i = l, j = r, pivot = a[(l + r) >> 1];
    while (i <= j) {
        while (a[i] < pivot) i++;
        while (a[j] > pivot) j--;
        if (i <= j) swap(a[i++], a[j--]);
    }
    quickSort(a, l, j);
    quickSort(a, i, r);
}
```


在算法竞赛中：
排序是“常规操作”，std::sort() 是“标配武器”。
如果不是题目要求手写算法，请直接使用 STL 提供的排序函数。



## 二分查找


---

### 一、什么是二分查找

**二分查找（Binary Search）** 是一种在**有序序列**中查找目标值的高效算法。  
它的核心思想是：  
> 每次将查找范围缩小一半，直到找到目标值或确定目标不存在。

时间复杂度仅为 **O(log n)**，非常高效。

---

### 二、基本思想

假设有一个有序数组 `a[0 ... n-1]`，要查找某个目标 `x`：

1. 取中点 `mid = (l + r) / 2`
2. 如果 `a[mid] == x`，找到了；
3. 如果 `a[mid] < x`，说明目标在右半边 → `l = mid + 1`
4. 否则目标在左半边 → `r = mid - 1`
5. 不断缩小区间，直到 `l > r`。

---

### 三、最基本模板（整型数组）

```cpp
// 1、满足条件的最小值
while (l < r) {
    int mid = (l + r) / 2;
    if (check(mid)) r = mid;  // 满足条件，往左收缩
    else l = mid + 1;         // 不满足条件，往右
}
return l;

// 2、满足条件的最大值
while (l < r) {
    int mid = (l + r + 1) / 2;  // 注意 +1 防止死循环
    if (check(mid)) l = mid;    // 满足条件，往右收缩
    else r = mid - 1;           // 不满足，往左
}
return l;
```
